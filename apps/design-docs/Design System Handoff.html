§ 01

File Layout

Where the design system lives.

A small, deliberate set of files under apps/admin/. The agent should not add a components/ui-kit folder, a theme/ package, or a design-tokens.json generator. Two CSS files, one fonts module, one glyph component, three layout primitives.

apps/admin/ ├── app/ │ ├── globals.css — @theme tokens, base, reset │ ├── fonts.ts — next/font, sets --font-serif-loaded and --font-mono-loaded │ ├── layout.tsx — html/body class bindings, Page chrome │ └── (admin)/... — routes ├── components/ │ ├── primitives/ │ │ ├── status-glyph.tsx — ● ◐ △ ○ — the five status SVGs │ │ ├── page.tsx — <Page> / <Section> / <Column> │ │ ├── numeric.tsx — <Numeric> tabular-nums helper │ │ └── rule.tsx — <Rule> hairline (replaces <Separator>) │ └── ui/ — shadcn primitives (restyled, see §4) ├── lib/ │ ├── api-types.ts — generated from FastAPI OpenAPI │ ├── cn.ts — clsx + twMerge │ └── format.ts — date, duration, run-id formatters └── tailwind.config.ts — minimal; tokens live in CSS (v4)
All design tokens live in app/globals.css under @theme. There is no separate tokens.css file. Tailwind v4 reads @theme directly; an additional indirection layer buys nothing for a single-app monorepo.
§ 02

Tokens

One CSS file. Paste verbatim.

Tailwind v4's @theme block is the token system. The values below are the entire palette, type scale, radii, and motion constants for the application. Treat changes to this file as design decisions.

CSSapps/admin/app/globals.css
@import "tailwindcss";

@theme {
  /* —— Surfaces ——————————————————————————————————— */
  --color-paper:        oklch(97.5% 0.004 80);   /* off-white, warm */
  --color-paper-deep:   oklch(95.5% 0.006 80);   /* inset rails, code blocks */
  --color-rule:         oklch(86%   0.006 80);   /* structural hairlines */
  --color-rule-soft:    oklch(91%   0.005 80);   /* secondary hairlines */

  /* —— Ink ——————————————————————————————————————— */
  --color-ink:          oklch(22%   0.008 80);   /* primary text */
  --color-ink-mid:      oklch(42%   0.008 80);   /* secondary text */
  --color-ink-faint:    oklch(58%   0.006 80);   /* labels, metadata */
  --color-ink-ghost:    oklch(72%   0.005 80);   /* watermark, disabled */

  /* —— Accents — used sparingly ——————————————————— */
  --color-accent:       oklch(40%   0.085 248);  /* iron-gall blue · action / "now" */
  --color-anomaly:      oklch(46%   0.105 38);   /* oxide red · failures only */

  /* —— Type families (loaded by next/font in §3) ——— */
  --font-serif:         var(--font-serif-loaded), Georgia, "Times New Roman", serif;
  --font-mono:          var(--font-mono-loaded), ui-monospace, Menlo, monospace;

  /* —— Type scale — opinionated, do not extend ————— */
  --text-display:   6.5rem;     /* 104px · masthead only */
  --text-h1:        2.75rem;    /* 44px · section titles */
  --text-h2:        1.75rem;    /* 28px · subsection */
  --text-h3:        1.125rem;   /* 18px · run-in heading */
  --text-body:      1.125rem;   /* 18px · body / prose */
  --text-data:      0.875rem;   /* 14px · tabular */
  --text-meta:      0.6875rem;  /* 11px · caption / label, tracked */

  /* —— Radii — effectively zero ——————————————————— */
  --radius-none:    0;
  --radius-chip:    2px;          /* the only permitted radius */

  /* —— Motion — rationed (see §7) ————————————————— */
  --ease-out:       cubic-bezier(0.2, 0, 0, 1);
  --dur-disclosure: 120ms;
  --dur-focus:      80ms;
  --dur-pulse:      2400ms;

  /* —— Override shadcn defaults — see §4 —————————— */
  --radius:         0;            /* shadcn reads this */
}

/* —— Base —————————————————————————————————————————— */
html { background: var(--color-paper); color-scheme: light; }

body {
  font-family: var(--font-serif);
  font-feature-settings: "onum", "kern", "liga";
  font-optical-sizing: auto;
  font-size: var(--text-body);
  line-height: 1.62;
  color: var(--color-ink);
  -webkit-font-smoothing: antialiased;
  text-rendering: optimizeLegibility;
}

/* Numerics — apply to any element that carries data. */
.tabular { font-variant-numeric: tabular-nums; }

/* Reduced motion — global gate (see §7). */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0ms !important;
  }
}

Fonts

next/font, one module, two faces.

Source Serif 4 with the optical-size axis loaded; JetBrains Mono with ligatures intentionally disabled. Variables are wired through the root <html> element and consumed by the @theme block.

import { Source_Serif_4, JetBrains_Mono } from "next/font/google";

export const serif = Source_Serif_4({
  subsets: ["latin"],
  weight: "variable",
  axes: ["opsz"],             // load the optical-size axis
  variable: "--font-serif-loaded",
  display: "swap",
  fallback: ["Georgia", "Times New Roman", "serif"],

export const mono = JetBrains_Mono({
  subsets: ["latin"],
  weight: ["400", "500", "600"],
  variable: "--font-mono-loaded",
  display: "swap",
  fallback: ["ui-monospace", "Menlo", "monospace"],

§ 04

shadcn/ui Re-theming

New York, with the gloss turned off.

shadcn's "New York" defaults ship with soft radii, drop shadows, and slate borders. We strip all three and rewire the variables to our tokens. The primitives keep their accessibility behavior; they lose their visual personality.

The global override

Add this block to globals.css immediately after @theme. It rewires shadcn's CSS variables to point at our tokens and disables all shadow utilities.

CSSapps/admin/app/globals.css — shadcn override
/* —— shadcn variable rewire ————————————————————————— */
:root {
  --background:           var(--color-paper);
  --foreground:           var(--color-ink);
  --card:                 var(--color-paper);
  --card-foreground:      var(--color-ink);
  --popover:              var(--color-paper);
  --popover-foreground:   var(--color-ink);
  --primary:              var(--color-accent);
  --primary-foreground:   var(--color-paper);
  --secondary:            var(--color-paper-deep);
  --secondary-foreground: var(--color-ink);
  --muted:                var(--color-paper-deep);
  --muted-foreground:     var(--color-ink-faint);
  --accent:               var(--color-paper-deep);
  --accent-foreground:    var(--color-ink);
  --destructive:          var(--color-anomaly);
  --destructive-foreground: var(--color-paper);
  --border:               var(--color-rule);
  --input:                var(--color-rule);
  --ring:                 var(--color-accent);

/* Strip all shadows from shadcn primitives. We use hairlines. */

/* Floating elements (Dialog, Sheet, Popover, Tooltip): hairline border. */
[data-slot="dialog-content"],
[data-slot="sheet-content"],
[data-slot="popover-content"],

  background: var(--color-paper);
  border: 1px solid var(--color-ink);
  box-shadow: none;
  border-radius: 0;
}

Status Glyphs

One component. Five kinds.

The PDF's four glyphs plus the new N/A hairline, packaged as a single <StatusGlyph> component. Inherits currentColor. Never animates. Carries an ARIA label so screen-readers can read the status as a word.

TSXapps/admin/components/primitives/status-glyph.tsx
import { cn } from "@/lib/cn";

export type StatusKind = "complete" | "partial" | "anomaly" | "empty" | "na";

const LABEL: Record<StatusKind, string> = {
  complete: "Complete",
  partial:  "Partial",
  anomaly:  "Anomaly",
  empty:    "Empty",
  na:       "Not applicable",
};

export function StatusGlyph({
  kind,
  size = 14,
  className,
}: {
  kind: StatusKind;
  size?: number;
  className?: string;
}) {
  // Stroke widens below 14px to preserve perceived weight.
  const stroke = size < 14 ? 1.5 : 1.25;
  const tone = kind === "anomaly" ? "text-anomaly" : "text-current";

  return (
    <svg
      role="img"
      aria-label={LABEL[kind]}
      width={size}
      height={size}
      viewBox="0 0 16 16"
      className={cn("inline-block align-[-0.125em] shrink-0", tone, className)}
    >
      {kind === "complete" && <circle cx="8" cy="8" r="5" fill="currentColor" />}
      {kind === "partial" && (
        <>
          <circle cx="8" cy="8" r="5" fill="none" stroke="currentColor" strokeWidth={stroke} />
          <path d="M8 3 A 5 5 0 0 1 8 13 Z" fill="currentColor" />
        </>
      )}
      {kind === "anomaly" && (
        <path d="M8 2.5 L13.5 12.5 L2.5 12.5 Z" fill="none" stroke="currentColor" strokeWidth={stroke + 0.15} />
      )}
      {kind === "empty" && <circle cx="8" cy="8" r="5" fill="none" stroke="currentColor" strokeWidth={stroke} />}
      {kind === "na" && <path d="M4 8 L12 8" stroke="currentColor" strokeWidth={stroke} strokeLinecap="square" />}
    </svg>
  );
}

§ 06

Layout Primitives

Three components, one grid.

Every screen is composed of <Page>, <Section>, and <Column>. No container components beyond these. The gutter, the rhythm, the rule between sections — all encapsulated, all consistent across the application.

TSXapps/admin/components/primitives/page.tsx
import { cn } from "@/lib/cn";

// Top-level page wrapper. Holds the gutter and max-width.
export function Page({ children, className }: {
  children: React.ReactNode;
  className?: string;
}) {
  return (
    <div className={cn("mx-auto max-w-[1180px] px-[clamp(20px,5vw,88px)]", className)}>
      {children}
    </div>
  );
}

// Section. Numbered, kickered, titled. A semantic chapter.
export function Section({
  number, kicker, title, lede, children,
}: {
  number: string;            // "01", "02", …
  kicker: string;            // "Core Palette"
  title: React.ReactNode;
  lede?: React.ReactNode;
  children: React.ReactNode;
}) {
  return (
    <section className="border-t border-rule pt-[88px] pb-14 first:border-t-0" data-screen-label={`${number} ${kicker}`}>
      <div className="grid grid-cols-[88px_1fr] gap-6 items-baseline mb-12">
        <div className="font-mono text-[12px] tracking-[0.08em] uppercase text-ink-faint border-t border-ink pt-3.5 w-[60px]">
          § {number}
        </div>
        <div>
          <p className="font-mono text-meta tracking-[0.14em] uppercase text-ink-faint mb-1.5">{kicker}</p>
          <h2 className="font-serif font-normal text-h1 leading-[1.1] tracking-[-0.012em] text-balance">{title}</h2>
        </div>
      </div>
      {lede && (
        <p className="italic text-[20px] leading-[1.5] text-ink-mid ml-[112px] max-w-[68ch] mb-10 text-pretty">
          {lede}
        </p>
      )}
      <div className="ml-[112px] max-w-[880px]">{children}</div>
    </section>
  );
}

// Column. Constrains running prose to the 68ch measure.
export function Column({ children }: { children: React.ReactNode }) {
  return <div className="max-w-[68ch]">{children}</div>;
}

The Rule primitive

Replaces shadcn's <Separator>. Always 1px, always border-rule. Two variants: strong (ink) for chapter boundaries, soft (rule-soft) for adjacent-row separators.

TSXapps/admin/components/primitives/rule.tsx
export function Rule({ variant = "default" }: { variant?: "default" | "soft" | "strong" }) {
  const tone =
    variant === "strong" ? "border-ink" :
    variant === "soft"   ? "border-rule-soft" :
                              "border-rule";
  return <hr className={`border-0 border-t ${tone} my-0`} />;
}

§ 07

Motion

Three keyframes. No more.

Define the three permitted motions once, in CSS, and use them everywhere they belong. Reduced-motion is gated globally in globals.css (see §2); these animations are no-ops when the user prefers it.

CSSapps/admin/app/globals.css — motion block
/* —— Disclosure: opacity fade only, 120ms. Day-detail, log expand. */
@keyframes disclosure {
  from { opacity: 0; }
  to   { opacity: 1; }
}
.disclose {
  animation: disclosure var(--dur-disclosure) var(--ease-out) both;
}

/* —— Focus underline: grows left-to-right, 80ms. Inputs, buttons. */
@keyframes focus-grow {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}
.focus-ring::after {
  content: "";
  position: absolute;
  left: 0; right: 0; bottom: -1px;
  height: 1px;
  background: var(--color-accent);
  transform-origin: left center;
  transform: scaleX(0);
}
.focus-ring:focus-visible::after {
  animation: focus-grow var(--dur-focus) var(--ease-out) forwards;
}

/* —— "Now" pulse: 2.4s opacity breath. Today indicator only. */
@keyframes now-pulse {
  0%, 100% { opacity: 1;   }
  50%       { opacity: 0.6; }
}
.now-pulse {
  animation: now-pulse var(--dur-pulse) var(--ease-out) infinite;
}
ClassWhereForbidden
.discloseDay-detail panel reveal, log expand, dialog contentNever combine with height/transform transitions. Opacity only.
.focus-ringInputs, buttons, tabbable cellsNever replace the underline with a glow ring.
.now-pulseThe single "today" dot in the month gridForbidden on any other element. If you find it elsewhere, remove it.
Anywhere shadcn ships animate-in fade-in-0 slide-in-from-* (Dialog, Sheet, Popover): replace with .disclose. The slide-in transforms are explicitly forbidden by §05 of the visual direction.
§ 08

Numeric & Data

Numbers are instruments, not decoration.

Every number the user might compare gets tabular figures. Every identifier gets monospace. The <Numeric> primitive enforces both at once and pairs them with a small italic-serif unit.

TSXapps/admin/components/primitives/numeric.tsx
import { cn } from "@/lib/cn";

export function Numeric({
  value,
  unit,
  size = "data",
  className,
}: {
  value: string | number;
  unit?: string;
  size?: "data" | "feature" | "display";
  className?: string;
}) {
  const scale = {
    data:    "text-[14px]",
    feature: "text-[28px]",
    display: "text-[56px] tracking-[-0.01em]",
  }[size];
  return (
    <span className={cn("font-mono tabular text-ink", scale, className)}>
      {value}
      {unit && <span className="font-serif italic text-ink-faint ml-2 text-[0.4em]">{unit}</span>}
    </span>
  );
}

Formatters

Centralize date, duration, and run-id formatting in lib/format.ts. Component code should never call toLocaleString directly.

TSapps/admin/lib/format.ts
import { format, formatDistanceStrict } from "date-fns";

// "04:12 UTC" — always 24-hour, always UTC, always pad-zero.
export const fmtTime = (d: Date) => format(d, "HH:mm 'UTC'");

// "Wed · 06 May" — short weekday, dot separator, no comma.
export const fmtDayHead = (d: Date) => format(d, "EEE · dd MMM");

// "9.4 s", "12 m", "1 h 04 m" — one significant decimal under 10 s.
export function fmtDuration(ms: number): string {
  if (ms < 10_000) return `${(ms / 1000).toFixed(1)} s`;
  if (ms < 60_000) return `${Math.round(ms / 1000)} s`;
  // …minute / hour cases here…
  return formatDistanceStrict(0, ms);
}

// "run.18322" — never the raw UUID; the truncated form is the identity.
export const fmtRunId = (n: number) => `run.${n.toLocaleString("en-US")}`;

§ 10

Acceptance Checklist

When the first screen is done.

Run through this list before merging the first real screen. If any item is unchecked, the visual system is not yet bound — fix it before adding features.

  1. The page background is oklch(97.5% 0.004 80), not #FFFFFF Open DevTools, inspect html. The computed background-color should match the token, not browser default. computed: oklch(97.5% 0.004 80)
  2. Body text renders in Source Serif 4, not Times or Georgia Inspect any <p>. Computed font-family should resolve to the next/font generated family name, with Georgia as fallback. If you see the system serif, fonts.ts isn't wired into <html>.
  3. Zero rounded corners anywhere on the screen Visually scan. If any element has a soft corner, find it in DevTools (border-radius) and remove the class. Exception: 2px chips, if any.
  4. Zero box-shadows on any element DevTools → Elements → search styles for box-shadow. Should return zero matches outside the shadcn override (which sets none).
  5. Status appears as a StatusGlyph component, not Unicode in a span Search the codebase for the literal characters ● ◐ △ ○. They should appear only in this design document and in status-glyph.tsx. Anywhere else: refactor.
  6. The accent color appears at most 3 times on the visible screen Count them. "Today" indicator, an active interactive control, a link. If you see four or more, one of them is decoration — find it.
  7. The anomaly red appears only when something has actually failed If the screen is in a happy state and you see oxide red, it's wrong. The only exception is the anomaly glyph in the legend.
  8. Every numeric value in a table uses tabular-nums The "1" and "8" in a column should align vertically. If digits jitter between rows, you're missing font-variant-numeric: tabular-nums.
  9. No icon library is imported grep -r "from \"lucide-react\"" should return only the shadcn primitives' internal use (Dialog close, Select chevron). Application code: zero.
  10. prefers-reduced-motion turns the application static In DevTools rendering tab, emulate reduce. The "now" pulse should stop, dialog opens become instant. Verify by reloading with the setting active.
  11. Section starts have an 88px top margin and a 1px border-rule rule above them The <Section> primitive enforces this; any hand-rolled section should match.
  12. Prose holds to ~68 characters of measure Drop a 90-word paragraph into the body. Lines should break naturally at ~68 chars. If they go to the viewport edge, you're missing <Column>.

Read alongside Visual Direction v0.1. If the agent produces a screen that satisfies every item in §10, the visual system has bound; ship features from there. Push back on any rule that costs more than it earns.
